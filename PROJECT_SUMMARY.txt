------------------------------
The Realman Robotic_Arm library does NOT support macOS natively!

Error you'll see on macOS:
  NameError: name 'libname' is not defined

Reason: The library only includes binaries for Linux and Windows.

âœ… SOLUTION - Use Docker:
--------------------------
1. Install Docker Desktop for Mac
2. Build: docker-compose build
3. Run: ./run_docker.sh --read

ðŸš€ QUICK START (macOS with Docker):
------------------------------------
# Build the container
docker-compose build

# Test connection (replace IPs with your actual arm IPs)
./run_docker.sh --arm1-ip 192.168.1.18 --arm2-ip 192.168.1.19 --read

# Read joint states from both arms
./run_docker.sh --read

# Set joint positions (in degrees)
./run_docker.sh --set 0 30 -30 0 90 0 0 --arm 1 --degrees

# Get arm information
./run_docker.sh --info

ðŸ”§ TROUBLESHOOTING:
-------------------
1. "Cannot connect to Docker daemon"
   -> Make sure Docker Desktop is running

2. "Connection refused"
   -> Check arm IP addresses and network connectivity
   -> Ping the arms: ping 192.168.1.18

3. Slow performance on macOS
   -> This is normal with Docker. Consider Linux for production.
